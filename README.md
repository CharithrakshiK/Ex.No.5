# Aim

To test and compare how different prompt patterns (broad/naïve vs. structured/basic) perform across multiple scenarios. Analyze the quality, accuracy, and depth of generated AI responses.

# AI Tool Used
Google Gemini
# Definition of Prompt Types

Naïve Prompt: Simple, vague, or open-ended request without proper guidance.

Structured Prompt: Detailed, contextual, and precise instructions that guide the AI for clearer responses.

# Methodology

Identify multiple test scenarios: story generation, factual Q&A, summarization, and advice.

Prepare two prompt versions for each task (naïve vs structured).

Run the experiment and capture outputs.

# Evaluate responses on:
Quality: Clarity & structure
Accuracy: Correctness of information
Depth: Completeness & reasoning

# Analysis

Naïve prompts produce short, vague, and unfocused responses.

Structured prompts consistently provide clear, detailed, and accurate outputs.

For simple queries, naïve prompts may suffice; for complex tasks, structured prompts are preferred.

# Conclusion

Structured prompts improve clarity, accuracy, and depth.

Few-Shot and Chain-of-Thought patterns work best for complex reasoning or creative tasks.

Zero-Shot is acceptable for simple or well-known queries.

Combining patterns (CoT + Role-Based, Few-Shot + Template) provides superior outputs.

Effective prompting is both an art and a science—choose patterns based on task complexity, domain, and desired output.

# Result

The comparative analysis of naïve and structured prompts across multiple scenarios was successfully conducted, showing that prompt clarity and structure significantly enhance AI output quality.
